내부 클래스
- 클래스 안에 정의된 클래스

1. 인스턴스 내부 클래스
- 인스턴스 자원 비슷한 성격
(멤버 변수, 인스턴스 메서드)
- 외부 클래스가 객체가 되어야만 객체를 생성할 수 있는 클래스
-> 객체 생성과 상관없는 자원(정적변수, 정적 메서드는 정의 X - jdk15)
-> jdk16+, 정적 자원 정의 가능



2. 정적 내부 클래스
	(static)
	- 내부 클래스에 static이 추가된 상태
	- 정적 자원과 비슷한 성격
	(정적 변수, 정적 메서드)
	- 외부 클래스가 객체가 되는 조건과 상관 X, 
	- 일반 클래스와 차이가 많이 없다. 
	- 외부 클래스의 인스턴스 자원 접근 X
	
3. 지역 내부 클래스 
- 메서드 내부에 정의된 클래스
- 인터페이스, 추상 클래스 -> 객체가 될 수 있다.
객체가 되는 조건
1) 메서드 내부(지역) : 실행 중에는 객체 생성 될 수 있다.
2) 멤버 변수 
3) 미 구현된 메서드의 구현체를 정의 함으로써 객체가 됨 

-> 지역 내부에 정의된 클래스의 메서드에서 지역 변수를 사용 -> 스택에서 제거 X -> 지역변수를 상수화(final) - 데이터 영역
